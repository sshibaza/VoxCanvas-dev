# Remove Connected App from VoxCanvas Setup — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove the Connected App step from the VoxCanvas setup wizard and guide users to register the JWT public key directly on the Contact Center record instead.

**Architecture:** The wizard's Connected App step was misleading — `SF_CONSUMER_KEY` / `SF_USERNAME` are written to `.env` but never consumed by the runtime. The SCRT2 JWT is signed with `iss=orgId, sub=callCenterApiName` and validated against the public key on the Contact Center record. We remove the Connected App step, add a new step that displays `certs/jwt.pem` with a Copy button, and collect only the tenant fields actually used at runtime.

**Tech Stack:** Node.js + Express 5, Vite + vanilla JS frontend, Tailwind v4, jsonwebtoken.

**Testing approach:** No test framework exists in this project. Each task ends with concrete manual verification steps (curl commands, browser flows). Do not add a test harness — out of scope per spec.

**Reference spec:** `docs/superpowers/specs/2026-04-17-remove-connected-app-design.md`

## Task 2: Add `/setup/public-key` endpoint

**Files:**
- Modify: `src/server/routes/setup.js`

- [ ] **Step 1: Add new handler after `/setup/certificate/download`**

Insert this block immediately after the `router.get('/setup/certificate/download', ...)` block (around line 126 in the original file; exact location is "right before `router.post('/setup/complete', ...)`"):

```js
  router.get('/setup/public-key', (req, res) => {
    const jwtPem = 'certs/jwt.pem';
    if (!fs.existsSync(jwtPem)) {
      return res.status(404).json({
        error: true,
        code: 'CERT_NOT_FOUND',
        message: 'JWT certificate not generated yet. Run certificate generation first.',
      });
    }
    try {
      const pem = fs.readFileSync(jwtPem, 'utf-8');
      res.type('text/plain').send(pem);
    } catch (err) {
      res.status(500).json({
        error: true,
        code: 'READ_FAILED',
        message: err.message,
      });
    }
  });
```
